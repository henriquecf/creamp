Figure Image

Feature image placed on top section, named id="hero", is representing you as website owner. The image itself is adjusted by class="hero-figure" for medium to large screen width and class="hero-figure-res" for smaller screen width. See illustration below:

Medium to large screen width - >> 992px

Small screen width - >> 768px and < 992px

Extra small screen width - < 768px

It calls three different images to be shown so it is recommended to provide your own three different images for those arrangements. Edit index file as well as the style and set the rule of style as you need:

// find these lines in index.htm

...

<img class="hero-figure" src="img/yourimage.jpg" alt=""> // set your image here 
<div class="hero-figure-res"></div> // set image by changing background url in .figure-res 

...


// add the style on head element or change directly on style.css

.hero-figure {
	
  -- adjust your image dimension and position --
	
}

@media screen and (min-width: 768px) and (max-width: 991px) { // small screen size
  .hero-figure-res {
    --
    background: url( .. ) center; // set and center your image, recommended dimension 300 x 300
    --
  }
}

@media screen and (max-width: 767px) { // extra small screen size
  .hero-figure-res {
    --
    background: url( .. ) center; // set and center your image, recommended dimension 100 x 100
    --
  }
}

To change the background, you can edit class="hero".

// add the style on head element or change it directly on style.css

.hero {
  background: url(...); // set your background image
}

Gallery

The gallery feature is generated by mixitup plugin to filter the categories and magnific popup plugin to make lightbox image presentation. The mark-up is below:

// control for mixitup plugin
<div class="gallery-control">
  <ul class="list-inline list-unstyled">
    <li class="filter active" data-filter="all"><span>All</span></li>
    <li>|</li>
    <li class="filter" data-filter="cat-1"><span>Web</span></li> // define the category within data-filter attribute
    <li >|</li>
    <li class="filter" data-filter="cat-2"><span>Design</span></li>
    <li>|</li>
    <li class="filter" data-filter="cat-3"><span>Flash</span></li>
  </ul>
</div>

<div id="grid" class="row gallery-inner">
  <div class="col-md-4 col-sm-6 mix mix_all cat-1"> // place the category as needed for every item
    <div class="gallery-frame">
      <a class="image-link" href="img/gallery/gal-img-1.jpg"><img src="img/gallery/gal-img-1.jpg" alt=""></a> // the mark-up for magnific popup 
    </div>
  </div>
  <div class="col-md-4 col-sm-6 mix mix_all cat-2">
    <div class="gallery-frame">
      <a class="image-link" href="img/gallery/gal-img-2.jpg"><img src="img/gallery/gal-img-2.jpg" alt=""></a>
  </div>
  </div>
    <div class="col-md-4 col-sm-6 mix mix_all cat-3 cat-1">

        <a class="image-link" href="img/gallery/gal-img-3.jpg"><img src="img/gallery/gal-img-3.jpg" alt=""></a>
      </div>
  </div>
</div>


// to initialize lightbox image presentation, just follow this mark-up

  data-map-lng="2.352222"  // map longitude

></div>

Twitter Feeds

The twitter feeds widget implementation is generated by jason mayes's twitter fetcher script. To use, you have to firstly get your own widget id that can be found in twitter widgets page. Check the widget creation url and find the line like this " data-widget-id="393276115713654784" ". The numbers is your id.

After you get the id, place it on template's data-widget-id then specify the value of data-length to display the numbers of tweets that you want returned. The value must be between 1 to 20.

<div id="twitter-widget" class="twitter-widget" 
  
  data-length="3" // number of tweets you want returned
  data-widget-id="410264841895424000" // twitter widget id
  
></div>
